Make Your First API Call

Here's a complete example using Python to predict lead scoring:

import requests
import json

# Your API configuration
API_KEY = "your-api-key-here"  # Replace with your actual API key
BASE_URL = "{{ request.url_root.rstrip('/') }}"

# Set up headers
headers = {
    "X-API-Key": API_KEY,
    "Content-Type": "application/json"
}

# Example data for lead scoring
data = {
    "company_size": "Medium",
    "industry": "Technology",
    "budget": 50000,
    "engagement_score": 75,
    "source": "Website"
}

# Make the API call
response = requests.post(
    f"{BASE_URL}/api/predict/lead-scoring",
    headers=headers,
    json=data
)

# Check the response
if response.status_code == 200:
    result = response.json()
    print("✅ Success!")
    print(f"Prediction: {result['prediction']}")
    print(f"Confidence: {result['confidence']:.2%}")
    print(f"Score: {result['score']}")
else:
    print("❌ Error:")
    print(f"Status Code: {response.status_code}")
    print(f"Response: {response.text}")
Expected Response:
{
    "prediction": "High Quality",
    "confidence": 0.87,
    "score": 85.3,
    "model_version": "1.0",

}

Handle API Responses

All API responses follow a consistent format. Here's how to handle them:

Success Response (200):
{
    "prediction": "High Quality",           // The main prediction result
    "confidence": 0.87,                    // Confidence score (0-1)
    "score": 85.3,                         // Numerical score (varies by model)
    "model_version": "1.0",                // Model version used
    "timestamp": "2025-01-15T10:30:00Z"    // When prediction was made
}
Error Response (401, 400, 500):
{
    "error": "Invalid API key",
    "message": "The provided API key is not valid or has been deactivated"
}
Common Error Codes:
  • 401: Invalid or missing API key
  • 400: Invalid request data or missing required fields
  • 500: Internal server error (contact support)

Process Multiple Records (Batch Upload)

For processing many records at once, use our batch upload feature:

Prepare Your CSV File:
# Example CSV for lead scoring (lead_data.csv)
company_size,industry,budget,engagement_score,source
Small,Healthcare,25000,60,Email
Medium,Technology,75000,85,Website  
Large,Finance,150000,90,Referral
Small,Retail,10000,45,Social Media
Upload with Python:
import requests

API_KEY = "your-api-key-here"
BASE_URL = "{{ request.url_root.rstrip('/') }}"

# Upload file for batch processing
with open('lead_data.csv', 'rb') as file:
    files = {'file': file}
    data = {'prediction_type': 'lead-scoring'}
    headers = {'X-API-Key': API_KEY}
    
    response = requests.post(
        f"{BASE_URL}/api/upload-batch",
        headers=headers,
        files=files,
        data=data
    )

if response.status_code == 200:
    result = response.json()
    print(f"✅ Uploaded! Processing {result['predictions_count']} records")
    print(f"Download results from: {result['download_url']}")
else:
    print(f"❌ Upload failed: {response.text}")

Best Practices & Tips

Security
  • Keep your API keys secure and private
  • Use environment variables for API keys
  • Rotate keys regularly from the dashboard
  • Deactivate unused keys immediately
Performance
  • Use batch upload for multiple records
  • Implement proper error handling
  • Cache results when appropriate
  • Monitor your API usage
Rate Limits: Each API key has a rate limit of 1000 requests per hour. Contact support if you need higher limits.
